Production-Ready Tree Endpoint

One endpoint for containers, data types, items, and data item verifications

/api/items now supports layered pagination and deep filtering in one contract. Container browsing paginates containers, single-container type mode paginates data types, and item mode paginates data items while preserving tree shape for frontend rendering.

Items API

Base path: /izipublish/api/items

Request Parameters

Param Required Description
userAddressYesAddress used for accessible-container scope.
includeNoCSV of CONTAINER,DATA_TYPE,DATA_ITEM,DATA_ITEM_VERIFICATION. Smart defaults are applied if omitted.
containerIdNoIf set, switches to single-container mode.
dataTypeIdNoRestrict to one data type within a container.
dataItemIdNoRestrict to a specific data item.
dataItemVerificationIdNoFilter data item verifications to one record id.
dataItemVerificationVerifiedNotrue/false data item verification status filter.
domainNoOptional publish-domain scope.
pageNo0-based page index.
pageSizeNoItems per page (clamped server-side).

Pagination Behavior

  • No containerId: pagination level is container.
  • containerId + DATA_TYPE only: pagination level is data_type.
  • containerId + DATA_ITEM: pagination level is data_item.

Each response includes a meta object to make frontend paging decisions deterministic.

Frontend Integration Contract

  • Use meta.paginationLevel to decide which UI list owns the current page controls.
  • Use meta.totalPages + meta.hasNext for next/prev button state.
  • Render nodes from containers[] -> dataTypes[] -> dataItems[] -> dataItemVerifications[].
  • Respect meta.dataItemVerificationFilteredAfterPagination when verification filters are active.
